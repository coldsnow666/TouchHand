# 扑克短局摸打 MVP 设计案

## 1. 项目定位

暂定名：`摸一手`

一句话：

> 用扑克牌玩 2 人短局摸打。玩家每回合选择拿对手最新弃牌或摸暗牌，抢先凑出本轮指定的三张牌型。

核心卖点：

- 扑克牌型 + 麻将式摸打节奏。
- 2 人短局，1 局 2-4 分钟。
- 每张弃牌都可能喂给对手。
- 适合小程序：轻资源、短反馈、可排行、可分享。

## 2. MVP 验证目标

MVP 目标不是做完整商业游戏，而是验证：

1. 玩家能否在 30 秒内理解规则。
2. 每轮是否能在 30-60 秒内结束。
3. 玩家是否会产生核心情绪：
   - 我不该打那张。
   - 他肯定在做同花。
   - 我赌摸牌还是拿弃牌？
4. 小程序闭环是否成立：
   - 首局完成
   - 再来一局
   - 分享挑战
   - 排行榜比较
   - 局后广告触达

## 3. 核心体验

玩家 80% 时间只做两个动作：

1. 拿牌：拿对手最新弃牌，或从牌堆摸暗牌。
2. 弃牌：从手牌中打出 1 张，成为对手下一回合唯一可拿的牌。

核心循环：

```text
观察本轮目标
-> 看对手最新弃牌
-> 决定拿弃牌或摸暗牌
-> 判断是否胡牌
-> 不胡则弃出一张
-> 对手获得拿牌机会
-> 直到有人胡牌或流局
```

核心张力：

> 我既要凑自己的牌，又不能把对手要的牌打出去。

## 4. MVP 规则

### 4.1 对局结构

- 2 人对局。
- MVP 优先做人机对战。
- 整局由多个独立小轮组成。
- 先到 5 分的一方赢得整局。

### 4.2 每轮开始

每轮独立重置：

- 清空双方手牌、弃牌、废弃区。
- 半随机生成本轮目标牌型。
- 双方各发 5 张手牌。
- 每轮交替先手。
- 先手首回合只能摸牌。
- 允许先手首回合胡牌。

### 4.3 目标牌型

目标牌型采用半随机不重复：

- 不连续出现同一个目标。
- 每 4 轮尽量覆盖至少 3 种目标。

MVP 目标池：

| 目标 | 条件 | 分值 |
| --- | --- | --- |
| 对子 | 任意 2 张同点数 | 1 分 |
| 三条 | 任意 3 张同点数 | 2 分 |
| 三张顺子 | 任意 3 张连续点数 | 2 分 |
| 三张同花 | 任意 3 张同花色 | 2 分 |

说明：

- MVP 只检查是否满足目标。
- 不比较牌型大小。
- 不做额外加分。
- 不做同花顺、葫芦、四条等高级牌型。

### 4.4 玩家回合

玩家回合流程：

```text
1. 选择拿对手最新弃牌，或从牌堆摸一张暗牌。
2. 手牌从 5 张变为 6 张。
3. 如果 6 张中任意组合满足本轮目标，可以胡牌。
4. 如果不胡，必须打出 1 张牌。
5. 手牌回到 5 张。
6. 打出的牌成为对手下一回合唯一可拿的弃牌。
```

最新弃牌规则：

- 只能拿对手最新打出的那一张牌。
- 更早的弃牌不可拿。
- 如果玩家选择摸暗牌，对手最新弃牌失效，进入废弃区。

### 4.5 胡牌结算

- 只有胡牌时亮牌。
- 胡牌时亮出用于满足目标的牌。
- 满足目标即可得分。
- 不比较大小。
- 不额外加分。
- 无放炮惩罚。
- 胡牌后本轮立刻结束。

示例：

```text
本轮目标：三张顺子
玩家手牌中有：4S 5H 6D
玩家胡牌，得 2 分。
```

### 4.6 流局

- 每轮最多 12 次玩家行动。
- 双方合计行动达到 12 次后无人胡牌，则本轮流局。
- 流局无人得分。
- 进入下一轮。

## 5. 小程序 MVP 闭环

MVP 不是只做核心玩法，而是做一个最小可测试商业闭环：

```text
首次进入
-> 10 秒教学
-> AI 对局
-> 胡牌结算
-> 上传成绩
-> 展示日榜/好友榜
-> 分享战绩或每日挑战
-> 局后广告触达
-> 再来一局
```

### 5.1 登录

MVP 策略：

- 优先游客体验。
- 首局不强制授权。
- 进入排行榜、分享战绩、同步成绩时再触发平台能力。

### 5.2 排行榜

推荐指标：每日挑战分。

每日挑战规则：

- 每天固定同一套牌序、目标序列和 AI 难度。
- 玩家比谁更快、更少行动赢到 5 分。
- 排行榜优先做日榜。
- 好友榜作为小程序版本重点，但需要额外评估开放数据域接入成本。

排行榜排序建议：

1. 是否获胜。
2. 总分。
3. 行动次数更少者优先。
4. 用时更短者优先。

不建议 MVP 做：

- ELO。
- 实时 PVP 排名。
- 总积分肝榜。

### 5.3 分享

分享内容建议：

- 我今天 8 手赢下每日挑战，你能更快吗？
- 这局我被一张弃牌反杀了。
- 同一副牌挑战你。

分享落地：

- 分享链接进入今日挑战。
- 保持同一套每日牌序。
- 不要求被分享者直接对战分享者。

### 5.4 广告

MVP 广告只放在局后，不影响当局公平。

允许：

- 看广告获得皮肤币。
- 看广告解锁一次复盘提示。
- 看广告增加一次每日挑战重试次数。

不建议：

- 看广告重摸一张。
- 看广告撤回弃牌。
- 看广告直接加分。

原因：

- 这些会破坏排行榜公平性。
- 玩家会觉得胜负被商业化干扰。

## 6. AI MVP

第一版 AI 不追求强，只追求能让玩家体验核心循环。

AI 基础策略：

1. 优先判断自己是否能胡。
2. 如果对手弃牌能让自己离目标更近，则拿。
3. 如果不能明显提升，则摸牌。
4. 弃牌时优先弃掉与当前目标弱相关的牌。
5. 避免弃出明显能帮助对手的牌。

AI 不需要在 MVP 做：

- 复杂读牌。
- 多轮记忆。
- 欺骗策略。
- 动态人格。

## 7. UI 与操作

主界面优先展示玩法，不做重 landing。

对局界面信息层级：

1. 当前比分。
2. 本轮目标与分值。
3. 对手最新弃牌。
4. 摸牌按钮。
5. 玩家手牌。
6. 胡牌提示。
7. 本轮剩余行动数。

操作原则：

- 拿弃牌：点击对手最新弃牌。
- 摸牌：点击牌堆。
- 弃牌：点击或拖拽手牌到弃牌区。
- 胡牌：满足条件时出现明确按钮。

新手引导：

- 第一局只出现对子目标。
- 用高亮引导玩家完成：摸牌、弃牌、拿对手弃牌、胡牌。

## 8. MVP 不做内容

为了控制范围，MVP 不做：

- 实时 PVP。
- 多人房间。
- 复杂牌型大小比较。
- 放炮惩罚。
- 角色技能。
- 卡牌养成。
- 长线剧情。
- 装备系统。
- 完整皮肤商城。
- 复杂任务系统。

这些都必须等核心循环验证后再考虑。

## 9. 数据埋点

必须记录：

- 首局开始率。
- 首局完成率。
- 首局用时。
- 单轮平均行动数。
- 流局率。
- 胡牌目标分布。
- 玩家选择拿弃牌的比例。
- 玩家选择摸牌的比例。
- 再来一局率。
- 每日挑战参与率。
- 分享点击率。
- 广告展示率。
- 广告完成率。

关键判断指标：

| 指标 | 通过线 |
| --- | --- |
| 新玩家 30 秒内理解规则 | 70% 以上 |
| 首局完成率 | 75% 以上 |
| 再来一局率 | 35% 以上 |
| 单轮平均时长 | 30-60 秒 |
| 流局率 | 低于 25% |
| 玩家能说出失败原因 | 60% 以上 |

## 10. 原型里程碑

### Milestone 1：灰盒玩法

目标：

- 完整跑通 2 人 AI 对局。
- 支持 4 种目标牌型。
- 支持胡牌、流局、计分。

验收：

- 玩家能完整打完一局。
- 没有明显死局。
- 每轮不会拖太久。

### Milestone 2：小程序体验壳

目标：

- 首次进入。
- 10 秒教学。
- 对局结算。
- 再来一局。
- 每日挑战入口。

验收：

- 新用户无需解释能完成第一局。

### Milestone 3：商业闭环

目标：

- 日榜。
- 分享挑战。
- 局后激励视频入口。
- 基础数据埋点。

验收：

- 能追踪首局、复玩、分享、广告行为。

## 11. 主要风险

### 风险 1：玩法太像简化麻将

应对：

- 强调扑克目标牌型。
- 使用 2 人短局。
- 不做吃碰杠。
- 不做复杂番型。

### 风险 2：三张目标太容易胡

应对：

- 控制首回合胡牌率。
- 增加指定目标牌型作为后续难度。
- 调整初始发牌算法。

### 风险 3：玩家不愿意读对手意图

应对：

- 用 UI 明确提示对手拿了哪张牌。
- 在局后复盘展示“你打出的这张让对手胡牌”。
- 新手局降低目标复杂度。

### 风险 4：排行榜不公平

应对：

- 每日挑战固定牌序。
- 广告不影响当局胜负。
- 排行榜只统计无广告干预的成绩。

## 12. 后续扩展

只有 MVP 数据通过后再考虑：

- 好友房。
- 实时 PVP。
- 3-4 人模式。
- 指定对子、指定同花、同花顺等高级目标。
- 角色轻技能。
- 皮肤和桌面主题。
- 复盘系统。
- 赛季挑战。

## 13. 当前结论

这个 MVP 的重点不是证明它像不像麻将，也不是证明它有多少系统。

它只需要证明一件事：

> 玩家会不会因为自己打出的一张牌、拿错的一张牌、错过的一张弃牌，而想立刻再来一局。

如果这个情绪成立，项目值得进入小程序完整体验开发。
